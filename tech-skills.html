Introduction & Technical Skills

An introduction to the skills I have aquired over the years as a Front End Web Developer.

Introduction

I began my career over a decade ago by completing an OCN Level 3 Web Development course, and volunteering at a local web development agency. From there I got my first job as a junior developer and progressed on to work for an agency in Cambridge to become lead Front End Developer.

My Philosophy

My approach to development is this: I try to avoid using third-party plugins or frameworks as much as possible and code as much as possible myself using native technologies. I feel this approach vastly improves my skill as a developer. The argument for using third-party systems is often that it helps teams of developers working on large projects - but personally I feel the opposite, that becoming too dependent on third party systems leaves projects and companies vulnerable to that third-party system becoming outdated or redunant - whereas the code and programming languages themselves stand the test of time. Javascript is still going to be here in 10 years time, but that fancy new framework might not be. It's a better investment in the long run simply to learn the tools of the trade!

A website should first and foremost present information in a way that is accessible to all. Pages should load quickly.

From a code perspective, I try to code a project in such a way that if another developer were to come along and take over the project in future, they would understand how it works. This often means avoiding trying to look clever by making things overly complex. I would much prefer to create a project that uses some native technologies and assemble them in a way that other developers could easily understand (including myself when I revisit the project in future!). I like projects that are simple and stand the test of time, rather than complex and fad-driven.

Technical Overview

I am a believer in the separation of concerns - HTML for document structure, CSS for presentation, Javascript for functionality.

HTML: With many of the website projects I work on, I deliberately opt to 'disable styles' in a browser from time to time and check if the website could be fully usable without any CSS present. The purpose of this exercise is not to assume that users would literally browse with styles disabled, but rather to encourage writing good quality, semantic markup. Progressive enhancement as a build philosophy states that the core of your application should be bullet proof and work across any device. So it's important that the HTML is well written- including features such as ARIA roles & landmarks for screen readers. I like to ensure that my HTML is also neatly indented and humanly readable in the code editor.

CSS: CSS has undergone some very exciting developments over the last few years - with flexbox and CSS grid, CSS custom properties, calc functions, clamp, as well as upcoming features like container queries - there has never been a better time to make full use of the capabilities of CSS.

I subscribe to the BEM (Block Element Modifier) approach to coding my projects and tend not to use other approaches such as utility classes (like those used in Tailwind).

I use SASS as a CSS preprocessor as part of a GULP task runner in a commercial setting. For personal projects I prefer to use raw CSS - as I enjoy the simplicity of being able to jump right into the CSS without running terminal commands or installing packages.

Javascript & AJAX: I code my projects in vanilla Javascript and have made it a priority over the years to specialise in being a vanilla Javascript developer, as opposed to using templating languages like jQuery. I am not opposed to JS frameworks, but I find that writing Javascript on projects allows me to gain a much deeper appreciation and learning of the core Javascript language - I feel I might not have gained so much knowledge if I had used lots of frameworks without a true understanding of what is going on 'under the hood'.

I can comfortably write Javascript including traversing the DOM, for loops, Event bubbling, Adding / Removing class based upon scroll position, Adding and removing cookies, changing HTML attributes, Change content based upon date/time, Toggles (e.g. Mobile Menu Toggle), arrays, functions, name spacing, event listeners, form control, using intersection observer to trigger events and animations on page scrolls and much more. I have experimented with writing games such as Space Invaders using raw Javascript. I'm continually learning new things and it is my goal to continue to expand and grow in this area.

SVG's: I have experience of directly manipulating inline SVG's to animate them on a page as well as using Javascript to manipulate page elements when hovering over certain SVG group areas.

Task Runners (e.g. Gulp): When I write HTML, SASS and Javascript- I use a task runner- in my case 'Gulp' - to watch and compile the code. I also install a plugin in my browser called "Live Reload" which allows me to see my changes happen live in the browser as it automatically refreshes each time I make a code update.

GIT Version Control: From local to staging to production. Using GIT to administer version control of projects. Experience of using Springloops and Buddy.

Terminal: Commands to push / pull / commit changes, resolve conflicts and merge changes, create new branches, switch branch etc.

CMS Integration: Wordress, Expression Engine, Craft CMS. I custom code templates to create themes using PHP, Twig or ExpressionEngine syntax. In the case of Wordpress, which I use most commonly, I tend to work with Advanced Custom Fields. I always aim to build a CMS setup that is tailored to a specific client and is as user friendly as possible.

Accessibiliy: A11y practices, page landmarks, keyboard friendly site navigations, contrast ratios, Progressive Enhancement build philosophy.

Testing & Optimising: Before “go live” I follow a process of looking at things such as Image optimisation, caching and use of tools such as Google Page Speed Insights to improve page loading times. I also ensure the groundwork for good SEO has been laid.

Email Marketing Campaigns: MJML, Foundation, Pardot, Mailchimp, Litmus testing

Static Site Generators: I have experience with two static site generators - Jekyll and HUGO. There are an endless list of static site generators to use, but I have found these two in particular to be very versatile.

Setting websites live: Using services such as TSO Host, Digital Ocean, GoDaddy and HostGator, editing hosts file via Terminal, creating Databases, importing databases, implimenting 'go live' checklists, pulling changes to live websites using GIT.

Operating Systems: Mac OS (work), Linux Ubuntu (personal), Linux Mint, Windows 7/10.

Sublime Text Snippets: I have written 'snippets' in sublime text which enable you to create a shortcut code, press tab and generate a ready to use code snippet.

Shell Scripting: I have written a number of BASH scripts for projects which incorporate Linux utilities such as Rsync, Sed, Grep, Head, Tail, PdfImages, ImageMagick to help drastically increase the speed of lengthy tasks and develop a system of efficient task automation.

Video & Audio: As a personal hobby I have experience with converting old analog video formats such as VHS tapes and 8mm video to digitised formats using hardware such as AV to HDMI converters & upscalers, video capture devices and software such as virtualdub, OBS, winDV and others. I have a working knowledge of the various correct resolutions for capturing footage depending on the source (e.g. PAL vs NTSC), as well as deinterlacing, compression algorithms. I have also had experience of using Audacity to convert mono sound into stereo sound.
