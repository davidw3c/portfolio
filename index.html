My Philosophy

I consider myself to be a digital minimalist - I think that too much of the internet has become bloated and confusing for users. A website should first and foremost present information in a way that is accessible to all. Pages should load quickly. The user experience should not be undermined by Javascript that hijacks a users ability to control the page or excessive distracting animations.

From a code perspective, I try to code a project in such a way that if another developer were to come along and take over the project in future, they would understand how it works. This often means avoiding trying to look clever by making things overly complex, or adopting cutting edge technology for the sake of doing so. I would much prefer to create a project that uses some native technologies and assemble them in a way that other developers could easily understand (including myself when I revisit the project in future!). I like projects that are simple and stand the test of time, rather than complex and fad-driven.
