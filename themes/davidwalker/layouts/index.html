What about React, Angular, Python, Ruby ect?

There are a multitude of other skills web developers are encouraged to learn these days. In the Javascript realm you have React, Angular, Vue, Next and many more. On the server side there are endless Content Management Systems and a choice of languages such as Python, Ruby, Java, Haskell and so on.

There is little doubt that these technologies have some incredible benefits and I do not rule out learning more about any of them. However, with only so many hours in the day- developers must make an educated decision about how to invest their time and energy.

One such way to gauge which technologies to learn is to look at what is popular. Technology Blogs, forums and online developer course schools create a lot of buzz surrounding certain trending frameworks or languages, but could it be that this can sometimes create misleading perceptions about what technologies are actually used in the real world?

According to data compiled from w3techs.com (as of January 2021) of the websites it monitors:

  • Wordpress is used on 39.5% of websites
  • PHP is used on 79.1% of websites
  • Javascript is used on 97.1% of websites

Meanwhile, perhaps surprisingly given their popularity on tech blog and forums:

  • Vue JS is used on 0.5% of websites
  • Angular JS is used on 0.4% of websites
  • React JS is used on 0.3% of websites
  • Ruby is the backend language of choice for 4.3% of websites
  • Python is the backend language of choice for 1.4% of websites

I have an enormous passion for learning and will absolutely continue to learn trending technologies, however my primary tech stack has been adopted for pragmatic purposes above all else - I want to be skilled in the technologies that have wide adoption rates and then add niche technologies to my stack as a complementary feature where possible.
